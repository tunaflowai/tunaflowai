# TunaFlowAI v0.5 Upgrade Notes

This upgrade turns TunaFlowAI into a stronger production-maturity foundation. It does not make the project a finished autonomous SaaS product, but it adds the safety and extensibility layers required before broader automation.

## Added

- Agent identity manager: user-defined name, display name, pronouns, biography, tone, traits, language, and boundaries.
- Sandbox runner for `run_command`: local, dry-run, and Docker modes with allowlists, deny patterns, timeout clamps, output limits, and sanitized environment.
- Provider HTTP integration tests for OpenAI-compatible, Gemini, and Anthropic providers.
- Real channel adapter foundations for Telegram, Discord, Slack, WhatsApp Cloud, and generic webhooks.
- Channel signature verification helpers.
- Skill trust registry for signing and verifying local job skills.
- Dashboard session auth with signed cookies when `TUNAFLOW_DASHBOARD_PASSWORD` is set.
- Task graph and per-task budgets.
- Browser operator foundation with approval-gated HTTP mutation tool.
- Local encrypted secrets vault.
- Policy-as-code engine.
- Market, server-health, schedule, and webhook observer foundations.
- New job skills and personas for finance, F&B, content, back-office admin, customer service, and DevOps.

## Identity vs Persona

Identity is the agent's stable name and personality. Persona is the current work role.

Example:

```text
Identity: Anna, warm and precise
Persona: Financial Market Analyst
Skills: financial-market-analyst, daily-reporter
```

Identity never grants permissions. Persona never bypasses policy. Skills never grant tool access.
